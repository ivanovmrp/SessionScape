# Experience design

## Primary journey

1. The therapist starts from a theme or a blank session.
2. They set a goal, duration, pace, pressure range, focus areas, and atmosphere.
3. They review client-specific preferences and exclusions.
4. The app produces a concise session blueprint and room-preparation checklist.
5. Before beginning, the therapist completes consent and comfort prompts.
6. Afterwards, they record optional feedback and save an improved version for future use.

## Core screens

| Screen | Job | Critical interaction |
| --- | --- | --- |
| Home | Re-enter work quickly | Continue draft or start from a theme. |
| Theme library | Find inspiration | Filter by goal, atmosphere, pace, pressure, and duration. |
| Session Builder | Create a complete plan | Live preview updates as choices change. |
| Client preference card | Honor stated preferences | User sees only the minimum needed for this session. |
| Preparation mode | Ready the room | Checkable, printable, no-distraction checklist. |
| Post-session reflection | Improve repeatability | Separate “what worked” from health/clinical records. |

## UX and content rules

- Use sensory but professional language: “Warm Radiance,” not terms that imply sexual service.
- Say “intended experience” or “may help the client feel,” not disease or treatment claims.
- Surface an unscented option wherever aroma appears.
- Sensitive-area work is never preselected. It requires an explicit, documented-in-the-moment consent checkpoint and appropriate draping reminder.
- Make client opt-out and therapist override possible from every builder step.
- Design the preparation view for a phone in a dim, calm treatment room: large tap targets, little prose, high contrast.
